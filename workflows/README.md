# Workflows

Plain-language, step-by-step instructions for each repeatable task.

Each workflow should cover:
- **Goal** — what this task accomplishes
- **Inputs** — what's needed to run it
- **Steps** — which tools to run, in what order
- **Output** — what the result should look like
- **On failure** — what to do when something goes wrong

If a task gets repeated more than once, give it a workflow. Don't rewrite or
delete workflows without the boss's permission — they're institutional knowledge.
